### 1. Nine Pillars Understanding

**Q1: Why is using AI Development Agents (like Gemini CLI) for repetitive setup tasks better for your growth as a system architect?**  
**Answer:**  
AI-native development mein hum specification-first approach use karte hain.Hum pehle specs likhte hain, phir AI agents implementation banate hnn aur hum validate karte hain. Is tarah, repetitive  tasks AI agents sambhalte hain, aur hm apna time architecture aur design thinking pr laga sakte hain. 

**Q2: Explain how the Nine Pillars of AIDD help a developer grow into an M-Shaped Developer.**  
**Answer:**  
In pillars ki wajah se developer ki skill set sirf ek jagah tak limited nahi rehti. Wo system architect, specification engineer, validator aur orchestrator ban sakte hn. Is tarah se wo ek M-Shaped Developer kehlate hn, jo multiple domains yani design, spec, architecture ko achi trh se smjh skte hn.

### 2. Vibe Coding vs Specification-Driven Development

**Q1: Why does Vibe Coding usually create problems after one week?**  
**Answer:**  
Vibe coding means k bs idea aya or code likhna shuru krdya, jis k back pr koi planning nahi hoti he, is se hm temporary task to generate kr skte hn lkn agr hm long term view pr dekhen to code me masly ajate hn or sara code messy hojata he.

**Q2: How would Specification-Driven Development prevent those problems?**  
**Answer:**  
Specification-Driven Development (SDD) mein sabse pehle structured spec likhi jaati hai, jo inputs, outputs, constraints, aur acceptance criteria define karta hai. Phir AI agents is spec ko follow karte hue code generate karte hain, aur us code ko spec ke mutabiq validate kiya jaata hai. Is approach se miscommunication kam hoti hai, aur future changes me bhi spec ko update karke easily handle kiye ja sakte hain.

---

### 3. Architecture Thinking

**Q1: How does architecture-first thinking change the role of a developer in AIDD?**  
**Answer:**  
Traditional development mein developer ka kaam mostly typing ya bugs fixing me hota tha, lekin AI-native development mein developer ka role transform ho kar architect aur validator ban jaata hai, yani developer ab sirf code likhne wala nahi, balki system ko design krta he, wo specs banata hai, AI agents ko batata hai ke kya build karna hai, aur phir un agents ka kaam check kar ke final design decide karta hai.

**Q2: Explain why developers must think in layers and systems instead of raw code.**  
**Answer:**  
Spec-first aur architecture-first soch se hm ek structured aur layered system tayar karte hn. Specifications act karte hain ksi agreement ki tarah jahan se AI components coordinate kar sakte hain.Is tarah agent orchestration aasan hoti hai, aur parallel development possible hoti hai multiple agents different task pr kam krte hn, jisse system zyada scalable aur maintainable banta hai.
